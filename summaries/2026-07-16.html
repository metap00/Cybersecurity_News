Microsoft Patch Tuesday July 2026: Record 570 Flaws Patched, Two Zero-Days Actively Exploited
🔍 What happened: Microsoft issued its largest-ever security update — 570 CVEs patched in a single Patch Tuesday — including two zero-days under active attack: CVE-2026-56164 (SharePoint Server privilege escalation) and CVE-2026-56155 (Active Directory Federation Services privilege escalation), both confirmed exploited in the wild before today's patches.
👤 Who's affected: Every organization running Microsoft SharePoint Server (2016, 2019, Subscription Edition), ADFS, or Windows broadly; 59 of the 570 CVEs are rated Critical.
✅ What to do: Apply July 2026 Patch Tuesday updates immediately — prioritize CVE-2026-56155 (ADFS, CVSS 7.8) and CVE-2026-56164 (SharePoint) since attackers are already using them.
🎯 ATT&CK: [Privilege Escalation] T1068 · Exploitation for Privilege Escalation
Both SharePoint and ADFS zero-days are being exploited by attackers who have an existing foothold in victim networks to elevate from standard-user to administrator-level access, enabling full domain control.
▶ Full analysis & beginner explanation

Microsoft's July 2026 Patch Tuesday is the biggest in the company's history, with 570 CVEs addressed — crushing the previous record set just last month. The dramatic spike is attributed in part to Microsoft's expanded deployment of AI-assisted vulnerability discovery tooling, which surfaced hundreds of internally discovered flaws. Of the 570 CVEs, 59 are rated Critical; 48 of those are remote code execution issues.

The two actively exploited zero-days are both elevation-of-privilege flaws. CVE-2026-56155 in ADFS (CVSS 7.8) allows an attacker to gain administrator privileges over the identity federation service — potentially enabling single sign-on token forgery across an enterprise. CVE-2026-56164 in SharePoint Server (CVSS 5.3) allows a lower-severity privilege escalation that still gives attackers elevated access to SharePoint data and configuration. Microsoft's Detection and Response Team (DART) credited discovery of CVE-2026-56155, indicating they caught active exploitation during incident response. A third zero-day, CVE-2026-50656 ("RoguePlanet") in Microsoft's Malware Protection Engine, was publicly disclosed but is not yet confirmed exploited in the wild.

The sheer volume of this release presents a real challenge for patch management teams. Security professionals should triage ruthlessly: the two exploited zero-days are immediate priorities, followed by the 48 Critical RCE flaws that could allow remote code execution without authentication.

🎓 For beginners: Every second Tuesday of the month, Microsoft releases security fixes — this is called "Patch Tuesday." A "zero-day" means attackers found and exploited this vulnerability before Microsoft could release a fix. Today Microsoft patched a record 570 security holes, two of which were already being actively used by attackers to gain extra power inside corporate networks. Apply the updates as fast as you can.
CVE-2026-56155 (ADFS) · CVSS: 7.8/10 — High-severity EoP; CVE-2026-56164 (SharePoint) · CVSS: 5.3/10 — Medium-severity EoP; both confirmed exploited in the wild with no prior patch available.

SonicWall SMA1000 Zero-Days Chained in Active Attacks: CVE-2026-15409 (CVSS 10.0) + CVE-2026-15410
🔍 What happened: SonicWall confirmed two vulnerabilities in its SMA1000 enterprise remote-access appliances are being actively chained together in real attacks: CVE-2026-15409 (CVSS 10.0, unauthenticated SSRF) lets attackers tunnel to internal services, while CVE-2026-15410 (CVSS 7.2, post-auth code injection) then enables OS command execution.
👤 Who's affected: Enterprises using SonicWall SMA6210, SMA7210, and SMA8200v appliances as VPN/remote-access gateways; federal agencies face a July 17 CISA deadline to patch.
✅ What to do: Contact SonicWall Support immediately for hotfixes (v12.4.3-03453 or 12.5.0-02835); if patching is impossible, isolate the management console and restrict appliance internet exposure.
🎯 ATT&CK: [Initial Access] T1190 · Exploit Public-Facing Application
Threat actors exploit CVE-2026-15409's SSRF to pivot from the internet into the SMA1000's internally-accessible management console, then leverage CVE-2026-15410 to execute arbitrary OS commands — turning a VPN gateway into a network entry point.
▶ Full analysis & beginner explanation

Rapid7's Managed Detection and Response (MDR) team discovered these two vulnerabilities being exploited in tandem in customer environments before SonicWall had patches available — making them true zero-days. CVE-2026-15409 is a Server-Side Request Forgery (SSRF) vulnerability in SMA1000's Work Place web interface that allows an unauthenticated remote attacker to force the appliance to make HTTP requests to internal services that should not be reachable from the internet. Crucially, this includes the appliance's own management console.

With access to the management interface established via the SSRF tunnel, attackers then exploit CVE-2026-15410, a code injection flaw in the Appliance Management Console that allows execution of arbitrary operating system commands. Together, these bugs give an unauthenticated internet attacker the ability to run commands on a device positioned at the network perimeter — ideal for establishing persistent backdoors or pivoting into the broader enterprise network.

SonicWall appliances have been a consistent target for nation-state and criminal threat groups, who prize edge devices because they offer persistent access with minimal endpoint agent visibility. CISA added both CVEs to its Known Exploited Vulnerabilities catalog on July 14, with federal agencies given until July 17 to remediate.

🎓 For beginners: A SonicWall SMA1000 is a "secure tunnel" device that lets employees log into company systems from home. Bug #1 (SSRF) tricks the tunnel device into thinking it's talking to an internal server — bypassing the "only employees can get here" protection. Bug #2 then lets attackers type commands as if they were the administrator. Chained together, an outsider can fully take over the VPN device without any credentials.
CVE-2026-15409 · CVSS: 10.0/10 — Critical unauthenticated SSRF in SMA1000 Work Place interface; CVE-2026-15410 · CVSS: 7.2/10 — Post-auth code injection enabling OS command execution on SMA6210, SMA7210, SMA8200v.

LegacyHive: Researcher Drops Unpatched Windows Privilege-Escalation PoC Hours After Patch Tuesday
🔍 What happened: Security researcher Chaotic Eclipse (MSNightmare) released a proof-of-concept for "LegacyHive," an unpatched privilege-escalation vulnerability in the Windows User Profile Service (profsvc) that grants a standard user privileged read-write access to other users' Windows Registry hives on all currently supported Windows desktop and server versions.
👤 Who's affected: All users of currently supported Windows desktop and server versions — the published PoC requires a second user's credentials, but the researcher states the full exploit does not.
✅ What to do: No patch exists yet — monitor Microsoft's advisories, apply principle of least privilege, restrict unnecessary local accounts, and watch for abnormal profsvc or registry activity.
🎯 ATT&CK: [Privilege Escalation] T1112 · Modify Registry
LegacyHive exploits a weakness in how profsvc loads user hive files to grant the attacker privileged write access to other users' Windows Registry hives — allowing persistent configuration changes, credential extraction, and further privilege escalation.
▶ Full analysis & beginner explanation

Roughly half an hour after Microsoft published its July 2026 Patch Tuesday disclosures, researcher Chaotic Eclipse (who goes by MSNightmare and is known for previous adversarial research against Microsoft) dropped details and a PoC for LegacyHive. The vulnerability targets the Windows User Profile Service (profsvc) and the way it loads "hive" files — segments of the Windows Registry that store a user's personal settings, application configuration, and environment variables.

By exploiting a flaw in how profsvc maps these hive files, an unprivileged attacker can gain privileged read-write access to the registry hives of other logged-in or previously-logged-in users. This is significant because registry hives can contain stored credentials, security settings, and persistence mechanisms for other software. The published PoC requires an additional user credential to function, but the researcher clarified publicly that their full private PoC does not — suggesting the real risk is higher than the published code demonstrates. The vulnerability affects all currently supported Windows versions.

No patch is available, and Microsoft has not yet published an advisory. The timing — dropped immediately after Patch Tuesday — is a deliberate pressure tactic used by some researchers to force vendors to expedite fixes by maximizing public exposure.

🎓 For beginners: Windows stores each user's settings in a file called a "registry hive." The User Profile Service (profsvc) loads that file when you log in. LegacyHive tricks profsvc into loading files in the wrong order, letting a standard user peek into and change another user's settings file — which can contain passwords and other sensitive data. Because there's no fix yet, this is particularly dangerous on shared Windows computers.
No CVE assigned yet · No CVSS score — Vendor has not issued a patch or advisory; exploitation status: PoC public, wild exploitation not yet confirmed.

SAP Patches CVSS 9.9 NetWeaver ABAP Memory Corruption Flaw That Risks Full System Compromise
🔍 What happened: SAP's July 2026 patch day addressed CVE-2026-44747 (CVSS 9.9), an out-of-bounds write vulnerability in NetWeaver Application Server ABAP (kernel versions 7.22–9.20) that allows a low-privileged authenticated attacker to corrupt memory and potentially achieve arbitrary code execution with complete compromise of confidentiality, integrity, and availability.
👤 Who's affected: Large enterprises worldwide running SAP NetWeaver AS ABAP for ERP, finance, HR, and supply chain operations — one of the world's most widely deployed enterprise platforms.
✅ What to do: Apply SAP Security Note 3747367 immediately; temporary workaround: disable ICF nodes with the specified property via transaction SICF in NetWeaver administration.
🎯 ATT&CK: [Lateral Movement] T1210 · Exploitation of Remote Services
CVE-2026-44747 enables an attacker holding even a low-privilege SAP account to send crafted ABAP requests that corrupt kernel memory — allowing code execution that crosses SAP trust boundaries and can compromise the entire ERP landscape from a single compromised credential.
▶ Full analysis & beginner explanation

SAP's July 2026 Security Patch Day included 16 new security notes, led by CVE-2026-44747 — an out-of-bounds write flaw in NetWeaver AS ABAP that earned a near-maximum CVSS score of 9.9. The vulnerability exists due to logical errors in memory management within the ABAP kernel; crafted requests can cause the kernel to write data beyond the boundaries of an allocated memory buffer, corrupting adjacent structures in a predictable manner.

The practical impact is severe: SAP's own advisory notes that successful exploitation results in "complete compromise of confidentiality, integrity, and availability" with "changed scope" — meaning an attacker can pivot beyond the initial SAP system to other connected SAP components and integrated systems. Only a low-privileged authenticated attacker is required, which is a low bar in many enterprises where broad user access to SAP is standard practice. CVE-2026-44747 affects ABAP kernel versions 7.22 through 9.20, covering installations spanning more than a decade.

SAP ERP systems are prime targets for sophisticated threat actors because they hold financial records, HR data, and supply chain intelligence. Security teams should treat SAP Note 3747367 as an emergency patch, not a routine update cycle item.

🎓 For beginners: SAP is software that large companies use to manage everything from payroll to factory operations. ABAP is the programming language that powers it. This bug means if you have even basic access to a company's SAP system, you can send a specially crafted message that corrupts the server's memory — potentially letting you read, change, or destroy any data in the entire SAP environment.
CVE-2026-44747 · CVSS: 9.9/10 — Critical out-of-bounds write in NetWeaver ABAP kernel 7.22–9.20; low-privilege attacker can achieve cross-boundary arbitrary code execution.

China-Aligned UNK_MassTraction Exploits Roundcube Webmail to Raid University Research Email
🔍 What happened: China-aligned threat cluster UNK_MassTraction has been systematically exploiting chained Roundcube vulnerabilities (CVE-2024-42009 XSS + CVE-2025-49113 server-side deserialization) since May 2026 to compromise university mail servers across the US and Canada, deploying webshells and VShell backdoors for persistent espionage access.
👤 Who's affected: US and Canadian universities — especially physics, astrophysics, particle physics, and engineering research departments with national security connections.
✅ What to do: Update Roundcube to the latest version immediately; audit mail servers for SquareShell PHP webshell and VShell Go-based backdoor indicators; restrict Roundcube admin panel to internal IPs only.
🎯 ATT&CK: [Collection] T1114.002 · Email Collection: Remote Email Collection
UNK_MassTraction chains a Roundcube XSS flaw (triggered by opening a crafted email) with a server-side deserialization bug to install webshells, granting persistent access to harvest research emails and attachments from compromised university mail servers.
▶ Full analysis & beginner explanation

UNK_MassTraction is a China-aligned espionage cluster with a documented focus on harvesting scientific research with national security implications. Since at least May 2026, the group has been targeting university physics and engineering departments across the United States and Canada using a sophisticated two-stage attack chain against Roundcube webmail servers.

The attack begins with a spear-phishing email containing a specially crafted payload that exploits CVE-2024-42009, a cross-site scripting (XSS) flaw in Roundcube. When a target opens the email in a vulnerable Roundcube client, malicious JavaScript executes in their browser, harvesting credentials and session tokens. The second stage exploits CVE-2025-49113, a server-side deserialization vulnerability, to install either SquareShell (a PHP webshell) or VShell (a Go-based backdoor) directly on the mail server. This combination gives attackers both immediate credential access and durable server-side persistence that survives password resets.

The targeting pattern — astrophysics, particle physics, and engineering with national security connections — aligns with well-documented Chinese state espionage priorities, mirroring historical campaigns such as those attributed to APT41 and related clusters that target advanced research institutions.

🎓 For beginners: Roundcube is open-source webmail software used by many universities. The first bug is triggered simply by opening a malicious email — no clicking links required — and runs attacker code in your browser. The second bug lets attackers install hidden "remote control" software on the mail server itself. Together they mean the university's email is fully compromised: attackers can silently read all emails without anyone knowing.

CVE-2026-45659 is a deserialization vulnerability in Microsoft SharePoint Server that allows any authenticated user with minimal permissions (Site Member or higher) to trigger remote code execution on the SharePoint server. Microsoft patched this flaw in May 2026 and originally assessed exploitation as "less likely" — a classification that proved optimistic, as CISA added it to KEV on July 1 following confirmed active exploitation.

The attack chain observed by defenders is particularly concerning: attackers are using SharePoint's broad enterprise footprint as an initial access point, exploiting CVE-2026-45659 to execute code on the server, then pivoting to deploy ransomware across the organization. SharePoint is deeply integrated into Microsoft 365 environments, meaning a compromised SharePoint server often provides attackers with a well-trusted network position from which lateral movement is straightforward. 💡 Today I Learned: Adversary-in-the-Middle (AiTM) Phishing ▶
AiTM phishing works by routing victims through an attacker-controlled proxy server that sits invisibly between you and the real login page — when you type your password and even your one-time MFA code, the attacker's server captures both in real time and immediately replays them to the real service to log in on the attacker's behalf, stealing your authenticated session cookie. Unlike older phishing that merely collected passwords for later use, AiTM kits steal the live authenticated session token, meaning the attacker gains access as if they were already fully logged in — defeating SMS codes, authenticator apps, and push notifications. The Jalisco and OmegaLord phishing kits reported this week are purpose-built AiTM tools targeting Microsoft 365, using device-code flow hijacking and real-time session relay to compromise enterprise accounts even when all standard MFA controls are enabled.
📖 Glossary: AiTM · EoP · Webshell · BEC · ABAP · VShell ▶
AiTM — Adversary-in-the-Middle; a phishing relay technique where an attacker's server sits between victim and real login service, stealing live session tokens to defeat MFA in real time.
EoP — Elevation of Privilege; a class of security vulnerability that allows a lower-privileged user or process to gain access rights beyond what was originally granted.
Webshell — A malicious script (often PHP or ASPX) uploaded to a compromised web server that gives attackers a persistent, browser-accessible remote command interface.
BEC — Business Email Compromise; fraud in which attackers impersonate executives, vendors, or partners via email to trick employees into authorizing wire transfers or disclosing sensitive data.
ABAP — Advanced Business Application Programming; SAP's proprietary programming language used in the NetWeaver Application Server to implement custom enterprise resource planning business logic.
VShell — A Go-language backdoor implant used by espionage actors for encrypted, persistent remote access to compromised mail and web servers, attributed to China-aligned UNK_MassTraction in current campaigns.
