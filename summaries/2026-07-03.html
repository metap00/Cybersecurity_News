CISA Adds SharePoint RCE Bug CVE-2026-45659 to KEV — Federal Agencies Must Patch by July 4
🔍 What happened: CISA confirmed active exploitation of a Microsoft SharePoint deserialization remote code execution vulnerability (CVE-2026-45659, CVSS 8.8), adding it to the KEV catalog on July 1 with a July 4 patch deadline for federal agencies — overriding Microsoft's own earlier "less likely to be exploited" assessment.
👤 Who's affected: Organizations running on-premises SharePoint Server 2016, 2019, or Subscription Edition; any authenticated user with as little as Site Member permissions can exploit it; SharePoint Online (Microsoft 365) is not vulnerable.
✅ What to do: Apply Microsoft's May 2026 SharePoint security update immediately; audit IIS application logs for deserialization errors or unauthorized code execution, and restrict SharePoint access to trusted accounts where possible.
▶ Full analysis & beginner explanation

CVE-2026-45659 is a deserialization vulnerability in Microsoft SharePoint Server that allows any authenticated user — including low-privileged accounts such as a standard "Site Member" — to execute arbitrary code on the SharePoint server. Microsoft patched the flaw in May 2026 but initially assessed exploitation as "less likely," a risk rating that CISA's July 1 KEV addition has directly contradicted with evidence of real-world attacks in progress.

The vulnerability stems from CWE-502 (Deserialization of Untrusted Data): SharePoint processes structured data packages from users without adequately validating their contents before reconstruction. An attacker crafts a malicious serialized payload and submits it to the server, which automatically "unpacks" it and runs the embedded instructions server-side. The consequences include data theft, ransomware deployment on the SharePoint server itself, or using it as a pivot point to reach internal network resources.

The low privilege requirement makes this especially dangerous: unlike vulnerabilities requiring administrative access, CVE-2026-45659 can be triggered with any valid user account. After a phishing email yields a standard SharePoint credential, an attacker has everything needed to execute code on the server. FCEB agencies face a July 4 deadline, reflecting CISA's assessment that active exploitation makes delay unacceptable.

🎓 For beginners: Serialization is like gift-wrapping data for storage or transport, and deserialization is the unwrapping on the other end. This vulnerability is like a mailroom that automatically opens every package without checking what's inside. Attackers send a "package" containing hidden instructions, and when SharePoint's mailroom opens it, those instructions run immediately with full server permissions — like finding a bomb inside a gift box that explodes the moment it's unwrapped.

ChocoPoC RAT Targets Security Researchers via Trojanized GitHub PoC Exploit Repositories
🔍 What happened: Attackers published at least seven fake proof-of-concept exploit repos on GitHub for high-profile CVEs, hiding a Remote Access Trojan called ChocoPoC inside Python dependency packages; the malware steals browser credentials and communicates back to attackers using Mapbox as a dead-drop command-and-control channel with domain fronting to evade detection.
👤 Who's affected: Security researchers, penetration testers, and developers who cloned and ran PoC exploits for CVEs in FortiWeb, PAN-OS, Ivanti Sentry, Check Point VPN, and Joomla SP Page Builder; YesWeHack and Sekoia reported the malware and its servers were still live as of July 1.
✅ What to do: Always run PoC exploit code in an isolated VM, never on a personal or work machine; audit Python packages in any exploit repos recently cloned, and check browser credential stores and network logs for anomalous Mapbox API traffic.
▶ Full analysis & beginner explanation

The ChocoPoC campaign, jointly disclosed by YesWeHack and Sekoia on July 1, 2026, targets security practitioners at the moment they are most eager to act quickly — downloading fresh exploit code to test hot new vulnerabilities. The attackers create GitHub repositories that convincingly imitate legitimate PoC exploits for trending CVEs, then hide the malware not in the main exploit file but in a Python dependency package that the PoC instructs the researcher to install — a technique designed to survive a quick code review of the main files.

Once installed, ChocoPoC harvests saved passwords, cookies, browser history, and autofill data from Chrome, Brave, Edge, and Firefox, and enumerates running processes, network configuration, and shell history. The attacker also receives a persistent remote shell for arbitrary command execution and file exfiltration. The campaign's most technically sophisticated element is its C2 channel: malware configuration is hosted as a dataset on Mapbox (a legitimate mapping-services provider), and communications are routed through DNS-over-HTTPS and domain fronting so that traffic resembles routine Mapbox API calls — difficult to detect or block via standard network controls.

Targeting security researchers is a high-value strategy: these machines often hold credentials for client environments, zero-day research notes, and access to restricted security infrastructure. The tactic mirrors prior nation-state campaigns (notably Lazarus Group operations in 2021 and 2024), though no attribution has been made for ChocoPoC. Any researcher who ran a PoC from GitHub in recent weeks should treat their machine as potentially compromised and initiate forensic review.

🎓 For beginners: Security researchers often download "proof-of-concept" code from GitHub to verify whether a vulnerability is real — like grabbing a lockpick guide to test your lock. ChocoPoC is like a booby-trapped guide: the guide itself looks legitimate, but one of the tools it asks you to install is secretly a spy that reads all your notes and passwords and phones them home. The spy then disguises its communications as normal map-loading requests so your security software doesn't flag it as suspicious.
💡 Today I Learned: Deserialization Attack ▶
Serialization is the process of converting program data — like a user profile or a document — into a flat stream of bytes so it can be stored or sent over a network, and deserialization is the reverse: reconstructing those bytes back into live program objects. A deserialization attack happens when a program automatically reconstructs a serialized payload without first verifying that its contents are safe, allowing an attacker to craft a malicious "package" that executes hidden instructions the moment the program unwraps it. Today's SharePoint vulnerability (CVE-2026-45659) is a textbook example: even a standard employee account can upload a poisoned serialized package to the server, which SharePoint trustingly unpacks — running the attacker's code with the server's full permissions before anyone can stop it.
📖 Glossary: RCE · XSS · EoP · HSIN · SAM · Dead Drop C2 · Domain Fronting · Deserialization · Phantom Squatting ▶
RCE — Remote Code Execution; a class of vulnerability that lets an attacker run arbitrary commands on a victim machine over a network, without requiring physical access or the victim to take any action.
XSS — Cross-Site Scripting; an attack where malicious JavaScript is injected into a trusted web page and runs inside another user's browser, enabling session cookie theft, credential harvesting, or page defacement.
EoP — Elevation of Privilege; an attack or vulnerability class that allows a low-privileged user to gain higher-level access (such as from a standard user account to SYSTEM or root) than they were authorized to have.
HSIN — Homeland Security Information Network; the U.S. government's secure web platform that connects over 120,000 federal, state, local, and private-sector partners to share real-time threat intelligence and coordinate emergency and event security operations.
SAM — Security Account Manager; the Windows database that stores hashed passwords for local user accounts, which attackers prize because gaining read access lets them crack or relay those credentials to move laterally across a network.
Dead Drop C2 — A command-and-control technique where malware reads its instructions from a legitimate, trusted third-party service (such as cloud storage, a mapping API, or a social media post) rather than connecting directly to an attacker-controlled server, making the traffic harder to detect or block.
Domain Fronting — A technique that disguises malicious network traffic by routing it through a high-reputation content delivery network (CDN) or cloud service, making command-and-control communications appear as legitimate requests to trusted services like Google or Mapbox.
Deserialization — The process of reconstructing a program object from a stored or transmitted byte stream; insecure deserialization occurs when this reconstruction happens without content validation, allowing attackers to embed executable instructions in the data that run automatically when unpacked.
Phantom Squatting — An emerging attack technique where adversaries register domain names that AI tools commonly hallucinate or suggest, then host phishing pages or malware at those addresses to capture users who follow AI-generated links to non-existent or invented URLs.
