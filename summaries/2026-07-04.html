New "Bad Epoll" Linux Kernel Flaw Lets Unprivileged Users Gain Root, Hits Android (CVE-2026-46242)
🔍 What happened: A use-after-free race condition in the Linux kernel's epoll subsystem allows any unprivileged local user to escalate to root, and can even be triggered from inside Chrome's renderer sandbox.
👤 Who's affected: All systems running Linux kernel v6.4 or newer (including Android Pixel 10 on kernel 6.6+) that have not yet backported the April 24, 2026 fix.
✅ What to do: Apply your distribution's kernel patch immediately; Android users should install OEM updates when released; check RHSA-2026:19568 for Red Hat guidance.
▶ Full analysis & beginner explanation

Bad Epoll was discovered and weaponized as a 0-day submission to Google's kernelCTF program, which offers $71,337+ for working Linux kernel exploits. The vulnerability is a race-condition use-after-free (UAF) in the epoll event notification subsystem — a core kernel interface used by virtually every networked Linux application. The race window is extraordinarily narrow (approximately six CPU instructions), but the researcher demonstrated reliable exploitation using goroutine scheduling tricks documented in the public proof-of-concept.

What elevates this from a dangerous LPE to a CRITICAL finding is the sandbox-escape capability: the exploit can be triggered from inside Chrome's renderer sandbox, which normally blocks access to nearly all kernel bugs. This creates a credible two-stage attack chain — a browser exploit lands the attacker in the sandboxed renderer, then Bad Epoll escapes the sandbox to root the entire OS. The kernel fix was merged on April 24, 2026, but many distributions are still shipping unpatched kernels, and a full Android root exploit targeting Pixel 10 (kernel 6.6) is reportedly still in development by the original researcher.

The flaw was introduced by a kernel commit from April 8, 2023 and affects any kernel version from v6.4 onward that hasn't backported the patch. Enterprise Linux users should check their distribution's security errata — Red Hat issued RHSA-2026:19568, and other major distributions have issued advisories.

🎓 For beginners: The Linux kernel is the core of the operating system — it controls who gets access to hardware, memory, and administrative privileges. The epoll system is how programs efficiently wait for network events, and this bug tricks it into accidentally handing an attacker root (administrator) access. What makes this especially dangerous is that it can be triggered from inside a web browser's protected bubble, meaning a malicious webpage could — in theory — result in complete control of the computer beneath it.

Microsoft Confirms "RoguePlanet" Defender Zero-Day (CVE-2026-50656) — SYSTEM Access on Fully Patched Windows, Still Unpatched
🔍 What happened: A TOCTOU race condition in Microsoft Defender's scanning engine lets a local attacker escalate to SYSTEM on fully patched Windows 10 and 11 — a public PoC exists and no patch is available as of July 4, 2026.
👤 Who's affected: All Windows 10 and Windows 11 users globally, with or without optional security updates applied through June 2026 Patch Tuesday.
✅ What to do: Monitor Microsoft Security Update Guide for a patch; restrict local user account privileges as a partial mitigation; consider additional endpoint detection until a fix ships.
▶ Full analysis & beginner explanation

RoguePlanet exploits a TOCTOU (Time-of-Check to Time-of-Use) race condition inside MsMpEng.exe, the core Microsoft Malware Protection Engine. Defender evaluates a file path to determine its safety, then reopens that same path later for analysis or remediation — but crucially, it does not hold an exclusive lock across both operations. An attacker can swap the file between these two moments, causing Defender (which runs with SYSTEM privileges to fight malware) to operate on attacker-controlled content and effectively hand over SYSTEM access.

What makes this story particularly significant is the human element: the researcher who discovered and publicly released this flaw, known only as "Nightmare Eclipse," did so deliberately as a protest against how Microsoft's Security Response Center (MSRC) handles bug bounty submissions. A working PoC has been publicly available for weeks. The flaw works regardless of whether Defender's Real-Time Protection is enabled — the Malware Protection Engine runs regardless. Microsoft confirmed the vulnerability and rated it "Exploitation More Likely" but has not shipped a fix as of today.

This is the second Defender zero-day released by Nightmare Eclipse in 2026, following BlueHammer (CVE-2026-33825). The pattern of repeated zero-days from a single researcher protesting disclosure practices is creating sustained risk for the entire Windows ecosystem while Microsoft manages the situation.

🎓 For beginners: Microsoft Defender is the built-in antivirus on Windows — it runs with the highest system privileges specifically so it can fight dangerous malware. This bug exploits that high privilege like a bait-and-switch: you hand Defender something harmless, it checks it and approves it, then before it can act you swap it for something malicious. Because Defender trusts itself implicitly, it carries out operations on the malicious content with full admin power, giving the attacker complete control of the PC.

👤 Who's affected: Any Windows user or organization that has not applied Microsoft's April 14, 2026 Patch Tuesday security update.
✅ What to do: Install the April 14, 2026 Microsoft security update immediately; audit all Windows endpoints for patch compliance; consider password rotation for local accounts on unpatched systems.
▶ Full analysis & beginner explanation

BlueHammer abuses "insufficient granularity of access control" in Microsoft Defender to grant local attackers read access to the SAM (Security Account Manager) database — the Windows file that stores NTLM password hashes for all local accounts. With those hashes in hand, attackers can either crack them offline to recover plaintext passwords, or use "pass-the-hash" attacks to authenticate as those users without cracking. In enterprise environments, this rapidly enables lateral movement across every machine sharing the same local admin credentials.

The CISA update explicitly labels BlueHammer as used in ransomware campaigns — a significant escalation from its initial KEV listing in April. Ransomware groups typically use BlueHammer-style LPE flaws as a post-initial-access escalation step: they gain a foothold via phishing or another vulnerability, then use BlueHammer to grab credentials, spread across the network, and deploy their encryption payload across hundreds of machines simultaneously. A patch has been available since April 14, 2026 — organizations running unpatched systems are at immediate risk.

Like RoguePlanet, BlueHammer was originally leaked publicly by the researcher "Nightmare Eclipse" as a protest against Microsoft's security response processes, before Microsoft had patched the issue. The public PoC remains freely available, making this accessible even to lower-sophistication attackers beyond the ransomware groups CISA identified.

🎓 For beginners: Windows stores scrambled versions of every local user's password in a locked file called the SAM database. BlueHammer picks the lock on that file, letting attackers grab all those scrambled passwords. Ransomware gangs use this to quickly collect every password across a company's computers, then spread their file-encrypting software to every machine at once — often locking up an entire organization within hours of gaining initial access. The fix has been out since April; there is no good reason to still be unpatched.

ToddyCat APT Deploys "Umbrij" Malware Abusing OAuth to Silently Access Corporate Gmail Accounts
🔍 What happened: The ToddyCat nation-state threat group deployed a new .NET backdoor called Umbrij that abuses Google's OAuth API to gain persistent, passwordless access to corporate Gmail accounts for espionage without alerting users.
👤 Who's affected: Organizations in Europe and Asia using Gmail for corporate communications, particularly those running Bitdefender ConnectAgent, Microsoft Visual Studio testing tools, or legacy Google Desktop Search software.
✅ What to do: Audit Google Workspace for unauthorized OAuth application grants; check for DLL sideloading artifacts in trusted process directories; review email forwarding rules for any newly added routes.
▶ Full analysis & beginner explanation

Umbrij, analyzed in depth by Kaspersky's Securelist team, represents a notable advancement in corporate espionage tradecraft. Unlike credential-theft attacks, it abuses Google's OAuth authorization framework — the legitimate system powering "Sign in with Google" — to obtain access tokens that grant persistent, silent read access to a target's email. The attacker never needs the user's password; once the token is obtained, it survives password changes. Kaspersky calls the specific technique "Shadow Token via Remote Debug" (STRD).

The initial infection uses DLL sideloading — the malware places a malicious DLL file with the same name as one expected by a trusted application (Bitdefender ConnectAgent, Microsoft VS testing tools, or discontinued Google Desktop Search). When a user launches the trusted program, it inadvertently loads and executes the attacker's code. The Umbrij .NET binary is then heavily obfuscated using ConfuserEx to defeat signature-based detection. From there, it silently registers an OAuth application with Google and grants itself access to the target's Gmail, harvesting strategic corporate communications.

ToddyCat has been active since at least 2020, primarily targeting government and military organizations across Europe and Asia. The shift to Gmail-hosted corporate email accounts indicates an evolving intelligence mandate focused on private-sector strategic planning. Organizations with high-value intellectual property, diplomatic communication, or defense supply chain roles are at particular risk.

🎓 For beginners: OAuth is the technology behind "Login with Google" — it lets apps access parts of your Google account with your permission. ToddyCat's malware tricks this system into granting permanent email access without you ever seeing or approving a prompt. The malware then hides inside trusted programs (like antivirus software) to avoid detection, quietly reading and forwarding your company's emails to foreign intelligence servers. Changing your password doesn't help — the OAuth token remains valid until explicitly revoked.

💡 Today I Learned: DLL Sideloading ▶
DLL sideloading is an attack technique where an adversary places a malicious file with the same name as a DLL (a shared code library) that a legitimate program expects to find in the same folder, so when you launch the trusted application, it accidentally loads and runs the attacker's code instead of the real library. Because the malicious code runs inside a trusted, well-known process, many security tools see only the trusted program's name and let it continue without raising an alarm. Today's ToddyCat/Umbrij campaign used DLL sideloading to hide inside Bitdefender ConnectAgent and Microsoft Visual Studio testing tools, allowing the malware to steal Gmail access tokens while appearing completely normal to endpoint security software.
📖 Glossary: CVE · KEV · OAuth · UAF · TOCTOU · APT · DLL Sideloading ▶
CVE — Common Vulnerabilities and Exposures; a standardized naming system that assigns each publicly known security vulnerability a unique identifier (e.g., CVE-2026-45659), enabling consistent tracking and communication across the industry.
KEV — CISA's Known Exploited Vulnerabilities catalog; a regularly updated list of vulnerabilities with confirmed real-world exploitation, used to prioritize patching — especially mandatory for U.S. federal agencies.
OAuth — Open Authorization; an industry-standard framework that allows applications to access a user's account data (email, files, contacts) with delegated permissions, without requiring the user to share their actual password.
UAF (Use-After-Free) — A class of memory corruption vulnerability where a program continues using a block of memory after it has been freed and potentially reallocated, allowing attackers to inject malicious data or code into that memory space.
TOCTOU — Time-of-Check to Time-of-Use; a race condition vulnerability where an attacker changes a resource (file, object, path) between the moment a program verifies it is safe and the moment the program actually uses it, causing the program to act on untrusted data.
APT (Advanced Persistent Threat) — A sophisticated, long-running cyberattack campaign typically conducted by nation-state actors or highly organized criminal groups, characterized by stealthy access, persistence over months or years, and focused intelligence or disruption objectives.
DLL Sideloading — A technique where attackers place a malicious DLL (Dynamic Link Library) file with the same name as one expected by a trusted application, causing the legitimate program to load and execute the attacker's code instead of the real library.
