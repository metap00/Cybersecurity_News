Azure CLI Password Spray Hits 78 Accounts in 81M+ Attempts — Chinese-Nexus Actor Bypasses MFA
🔍 What happened: A threat actor linked to Chinese-origin infrastructure (LSHIY LLC, AS32167) launched over 81 million automated login attempts against Microsoft Azure CLI between June 12–26, successfully compromising 78 accounts across 64 organizations by replaying old breach credentials through a deprecated OAuth flow.
👤 Who's affected: Any organization using Microsoft Azure CLI, especially those with legacy accounts holding passwords exposed in prior data breaches that were never rotated.
✅ What to do: Disable the ROPC OAuth flow in your Azure tenant, enforce Conditional Access Policies blocking legacy authentication, and audit sign-in logs for traffic from IPv6 range 2a0a:d683::/32.
▶ Full analysis & beginner explanation

Researchers at Huntress documented this ongoing campaign, tracked as LSHIY after the infrastructure provider supplying the IPv6 attack range (2a0a:d683::/32). The actor replays old username-and-password pairs exposed in prior data breaches, targeting the Resource Owner Password Credentials (ROPC) OAuth flow within Azure CLI — a deprecated authentication pathway that allows direct credential submission without triggering modern MFA or Conditional Access Policy checks.

Between June 12 and 21, the actor averaged two to four successful account compromises daily. The pace jumped dramatically on June 22, when 30 identities across 23 businesses were hit in a single day. Third-party telemetry consistently links LSHIY's IPv6 prefixes to Chinese-origin traffic; corporate records reference addresses in Hong Kong, Wuhan, and a New York shared office — though definitive nation-state attribution has not been confirmed.

The core weakness here is organizational hygiene, not a software bug: many enterprises have not disabled legacy OAuth flows or required users to rotate passwords exposed in prior breaches. ROPC was designed for backwards compatibility and explicitly bypasses phishing-resistant MFA, making even otherwise well-secured Azure tenants vulnerable if legacy authentication remains enabled in their policy set.

🎓 For beginners: Imagine someone found an old key to your house from a past break-in and tried it on your new lock — and it still worked. That's what's happening here: old passwords leaked in previous hacks are being tried against Microsoft cloud accounts using a backdoor login method that skips two-factor authentication entirely. The fix is to close that backdoor by disabling the old login method and making sure every account uses modern authentication that can't be bypassed this way.
No CVE — campaign exploits legacy ROPC misconfiguration, not a software vulnerability · Severity: CRITICAL — nation-scale credential spraying, active MFA bypass, confirmed account compromises at 64 organizations.

DuneSlide: Zero-Click Prompt Injection Flaws in Cursor IDE Let Attackers Escape Sandbox and Run Commands
🔍 What happened: Cato AI Labs disclosed two critical vulnerabilities (CVE-2026-50548/50549) in Cursor IDE that allow attacker-controlled content inside a code repository to silently escape the editor's sandboxing and execute arbitrary commands on a developer's machine — with zero clicks required.
👤 Who's affected: All developers using Cursor IDE before version 3.0 who open repositories or files containing untrusted or externally sourced content.
✅ What to do: Update Cursor IDE to version 3.0 immediately; treat AI-powered IDE agents with the same suspicion as untrusted user input when opening unknown repositories.
▶ Full analysis & beginner explanation

Cato AI Labs named this dual vulnerability chain "DuneSlide." Both CVE-2026-50548 and CVE-2026-50549 carry a CVSS score of 9.8 (9.3 under CVSS v4.0) and exploit prompt injection — the ability to embed adversarial instructions inside data that an AI model reads and then acts upon — to escape Cursor's sandboxed execution environment and execute arbitrary commands directly on the developer's host OS.

CVE-2026-50548 abuses a working-directory allowlist bug: when Cursor's AI agent sets a non-default working_directory on its run_terminal_cmd tool, Cursor automatically adds that path to the allowed-write list without verifying it's inside the project sandbox. An injected prompt can direct the agent to set this path to any system location. CVE-2026-50549 exploits a symlink-resolution fallback: when Cursor's safety check cannot resolve the real destination of a path (because the attacker makes a parent directory temporarily unreadable), Cursor trusts the shortcut's unverified in-project path instead — permitting writes outside the sandbox.

The zero-click nature is the critical differentiator. An attacker could embed malicious prompt instructions in a package README, a source file, or a data file inside a repository. When a developer opens that repository and the AI agent parses the file, the agent could download and execute a backdoor or exfiltrate SSH keys before the developer interacts with the editor at all. This is a supply-chain attack surface unique to AI-native development tools. Cursor 3.0 (released April 2) contains the fix.

🎓 For beginners: Cursor is a popular code editor with a built-in AI assistant. These bugs mean a hacker could hide secret instructions inside a code project — like hiding a message in a document saying "as soon as you read this, hand your house key to the stranger outside" — and the AI would literally follow those instructions, running harmful programs on the developer's computer before the developer clicks anything. The fix is simply updating the editor to version 3.0.
CVE-2026-50548 & CVE-2026-50549 · CVSS: 9.8/10 — Zero-click prompt injection to pre-authenticated RCE via sandbox escape in Cursor IDE; patched in Cursor 3.0.

CVE-2026-8037 lives inside the escape_quotes() function in Kemp LoadMaster's API handler — a function meant to sanitize user input before passing it to a shell command. A coding flaw causes it to fail to properly null-terminate sanitized strings, leading to an out-of-bounds read into adjacent heap memory. Attackers craft requests to the /accessv2 endpoint that inject shell commands via this uninitialized heap read, gaining root-level execution on the appliance with no authentication required.

The deployment position of Kemp LoadMaster amplifies the impact: as a load balancer, it sits at the outermost edge of enterprise networks — the first appliance to receive inbound internet traffic. A compromised appliance gives attackers traffic interception capabilities, a persistent network foothold for lateral movement, and a trusted internal IP from which to pivot deeper. Many enterprises have slow patch cycles for edge appliances because taking a load balancer offline requires coordinated traffic failover.

Langflow RCE CVE-2026-33017 Exploited to Deploy Monero Miner and SSH-Propagating Worm on AI Servers
🔍 What happened: Attackers are exploiting a CVSS 9.3 unauthenticated RCE flaw in Langflow's public build endpoint to deploy a Monero cryptocurrency miner that comes bundled with an SSH worm, which automatically spreads to every SSH-reachable host found in the victim's known_hosts file.
👤 Who's affected: Developers and organizations running Langflow AI orchestration platforms on versions prior to 1.9.0, particularly instances exposed to the public internet.
✅ What to do: Update Langflow to version 1.9.0 or later immediately; set AUTO_LOGIN=false and remove public internet exposure from Langflow instances.
▶ Full analysis & beginner explanation

CVE-2026-33017 is an unauthenticated RCE flaw in Langflow's POST /api/v1/build_public_tmp/{flow_id}/flow endpoint. The endpoint accepts an attacker-controlled "code" field and evaluates it as Python directly within the server process — with no authentication check whatsoever. Carrying a CVSS v4.0 score of 9.3, the flaw affects all Langflow versions before 1.9.0 and was being exploited in the wild within 20 hours of public disclosure.

TrendMicro's analysis of the attack reveals a bash dropper that creates a hidden persistence directory at /var/tmp/.xlamb/, downloads the main malware binary (lambsys.elf), and launches it as a background Monero cryptocurrency miner. The dropper doubles as an SSH-key-reuse worm: it enumerates ~/.ssh/known_hosts and available SSH agent sockets, then attempts to authenticate to every known host using the victim's own SSH credentials, propagating itself laterally across the victim's entire reachable infrastructure.

The self-propagating SSH worm component makes this particularly dangerous for cloud and shared environments. An attacker who gains entry through a single exposed Langflow instance can potentially reach internal servers, CI/CD pipelines, and cloud workloads that implicitly trust the compromised machine via shared SSH key rings — turning a single vulnerable endpoint into a broad infrastructure compromise.

The malware that gets installed does two things: it quietly uses your server's computing power to generate cryptocurrency for the attacker (costing you real money in electricity and cloud bills), and it secretly spreads itself to every other server your machine connects to via SSH — like a virus that copies itself from device to device across your whole network.

🔍 What happened: CISA updated its Known Exploited Vulnerabilities catalog to confirm ransomware groups are actively leveraging "BlueHammer" (CVE-2026-33825), a privilege escalation flaw in Microsoft Defender, to elevate access during attacks — months after a patch was first available.
👤 Who's affected: Windows users and organizations running Microsoft Defender on endpoints that have not applied the April 14, 2026 cumulative security update.
✅ What to do: Apply Microsoft's April 14, 2026 cumulative update for Microsoft Defender across all Windows endpoints immediately; verify patch deployment via your endpoint management tooling.
▶ Full analysis & beginner explanation

CVE-2026-33825, nicknamed "BlueHammer," is a local privilege escalation (LPE) vulnerability in Microsoft Defender stemming from insufficient access control granularity. An authenticated local attacker can exploit the flaw to elevate their privileges on a targeted Windows system. Microsoft released a fix on April 14, 2026, and CISA first added BlueHammer to its KEV catalog on April 22 after confirming in-the-wild exploitation. CISA's updated KEV entry now explicitly ties the flaw to ransomware campaigns.

LPE vulnerabilities like BlueHammer function as force multipliers during ransomware attacks. After achieving an initial foothold — via phishing, another vulnerability, or stolen credentials — ransomware operators use privilege escalation to gain SYSTEM or Administrator rights. From that elevated position, they can disable security tools, move laterally across the network, exfiltrate data for double-extortion leverage, and finally deploy ransomware payloads at scale. BlueHammer fits cleanly into this playbook.

The researcher behind BlueHammer's discovery, known publicly as Chaotic Eclipse and Nightmare Eclipse, disclosed the vulnerability after a public dispute with Microsoft's vulnerability handling process. This type of friction-driven full disclosure creates time pressure for defenders: attackers often monitor researcher disputes and capitalize on new technical details before organizations have finished patching.

🎓 For beginners: Microsoft Defender is the antivirus software built into Windows. BlueHammer is a bug that lets a hacker who has already gotten a foot in the door "level up" their access — like a visitor who snuck into the lobby finding a master keycard that opens every room. Ransomware gangs are using this as a stepping stone: they get a small initial foothold through another method, then use BlueHammer to take complete control of the machine before locking all files and demanding payment.

💡 Today I Learned: Prompt Injection ▶
Prompt injection is an attack where a malicious actor hides secret instructions inside text that an AI system will read, causing the AI to follow those instructions instead of its legitimate task — similar to slipping a forged note into someone's to-do list. These hidden instructions can appear anywhere the AI ingests untrusted data: documents, emails, web pages, code files, or even database records. Today's DuneSlide vulnerabilities in Cursor IDE are a real-world example: by hiding adversarial prompts in a code repository's files, an attacker could hijack the AI editor's actions and execute commands on a developer's computer without any user clicking anything.
📖 Glossary: ROPC · Password Spray · Prompt Injection · Sandbox Escape · MFA · CAP · CVSS · OWA · Cryptominer · Load Balancer · EEMS ▶
ROPC — Resource Owner Password Credentials, a deprecated OAuth flow that lets apps collect usernames and passwords directly, bypassing modern authentication controls like MFA.
Password Spray — An attack where a threat actor tries a small set of common passwords against a large number of accounts to avoid triggering lockout thresholds that would activate if many passwords were tried on one account.
Prompt Injection — An attack where malicious instructions embedded in data or documents manipulate an AI model into performing unintended or harmful actions instead of its legitimate task.
Sandbox Escape — A technique by which malicious code breaks out of a restricted execution environment (sandbox) designed to contain it, gaining access to the broader operating system.
MFA — Multi-Factor Authentication, a security mechanism requiring two or more verification factors (e.g., a password plus a one-time code) to prove identity, making stolen passwords alone insufficient for account access.
CAP — Conditional Access Policy, a Microsoft Azure control that enforces access rules (such as requiring MFA or blocking legacy auth) based on conditions like user location, device compliance, or sign-in risk.
CVSS — Common Vulnerability Scoring System, a standardized framework that assigns vulnerabilities a severity score from 0 to 10 (where 10 is most severe) to help organizations prioritize patching efforts.
OWA — Outlook Web Access, Microsoft's browser-based interface for reading and sending email on Exchange Server without installing the Outlook desktop application.
Cryptominer — Malware that secretly hijacks a victim's computing resources to solve cryptocurrency mining calculations for the attacker, costing the victim electricity, cloud compute costs, and system performance.
Load Balancer — A network appliance that distributes incoming traffic across multiple servers to improve performance and availability; often sits at the edge of enterprise networks, making it a high-value attack target.
EEMS — Exchange Emergency Mitigation Service, a Microsoft feature that automatically pushes temporary mitigations to on-premises Exchange Server when a vulnerability is discovered, bridging the gap before a permanent patch is released.
