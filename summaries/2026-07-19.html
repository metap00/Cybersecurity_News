Inc Ransomware Chains Two SonicWall SMA1000 Zero-Days for Root-Level Network Access
🔍 What happened: Inc ransomware actors are chaining two patched zero-days in SonicWall SMA 1000 Series remote-access appliances — a CVSS 10.0 unauthenticated SSRF (CVE-2026-15409) and a CVSS 7.2 post-auth code injection flaw (CVE-2026-15410) — to achieve root-level command execution and stage ransomware deployments across enterprise networks.
👤 Who's affected: Organizations running SonicWall SMA6210, SMA7210, or SMA8200v appliances that have not applied hotfix 12.4.3-03453 or 12.5.0-02835 are exposed and should treat any internet-facing SMA 1000 as potentially compromised.
✅ What to do: Apply SonicWall hotfix 12.4.3-03453 or 12.5.0-02835 immediately; rotate all credentials that may have transited these appliances; hunt for lateral movement indicators and ransomware staging activity in affected networks.
🎯 ATT&CK: [Initial Access] T1190 · Exploit Public-Facing Application
CVE-2026-15409 exploits the SMA1000 Work Place portal from the open internet, giving Inc ransomware an unauthenticated foothold before chaining to CVE-2026-15410 for full root-level control of the VPN gateway.
▶ Full analysis & beginner explanation

SonicWall's SMA 1000 Series appliances are internet-facing VPN and remote-access gateways used by enterprises to secure employee connections. Rapid7's Managed Detection & Response team discovered Inc ransomware actors actively chaining two vulnerabilities to gain unauthenticated root access: CVE-2026-15409 (CVSS 10.0) is a server-side request forgery flaw in the SMA1000 Work Place web portal that lets an attacker force the appliance to make requests to arbitrary hosts, establishing initial code execution. CVE-2026-15410 (CVSS 7.2) is a code injection vulnerability in the Appliance Management Console that escalates from that toehold to full root-level OS command execution.

Once inside at root, Inc actors sweep victim networks for credentials, conduct lateral movement, and prepare ransomware payloads. CISA added both CVEs to its Known Exploited Vulnerabilities catalog on July 14. SonicWall has released hotfixes and urges immediate patching; organizations should also rotate any credentials that may have been in scope on compromised appliances.

The chaining pattern here is notable: a CVSS 10.0 unauthenticated SSRF alone would not yield code execution, but combined with the authenticated code injection, attackers bypass the authentication requirement of the second bug by using SSRF to obtain enough internal context to satisfy it. This "vulnerability chaining" is increasingly common among sophisticated ransomware groups seeking guaranteed entry.

🎓 For beginners: Think of SonicWall's SMA appliance as the corporate front door for remote workers. CVE-2026-15409 lets an attacker knock on that door and trick it into opening a side window (SSRF — it makes requests on the attacker's behalf). CVE-2026-15410 then lets the attacker climb through that window and take over the entire building's computer system as an administrator. Once they have that level of control, Inc ransomware locks all your company's files and demands a ransom payment.

WordPress wp2shell (CVE-2026-63030): Unauthenticated RCE in Tens of Millions of Default Sites — Patch Now
🔍 What happened: Researchers disclosed CVE-2026-63030 ("wp2shell"), a critical pre-authentication remote code execution vulnerability in WordPress core 6.9–7.0.1 that chains REST API route confusion with SQL injection to let anonymous attackers execute arbitrary PHP code on any unpatched default installation.
👤 Who's affected: Any website running WordPress 6.9 through 7.0.1 — estimated at tens of millions of sites globally — is vulnerable, with no plugins or special configuration required for exploitation.
✅ What to do: Verify your WordPress version is 6.9.5, 7.0.2, or later; WordPress.org forced automatic updates for affected versions, but sites with auto-updates disabled must update manually via the dashboard or wp-cli.
🎯 ATT&CK: [Initial Access] T1190 · Exploit Public-Facing Application
wp2shell exploits WordPress's REST API batch endpoint — a public-facing web interface — requiring zero authentication to chain REST route confusion with SQL injection into full remote code execution on the underlying server.
▶ Full analysis & beginner explanation

WordPress powers an estimated 40% of all websites on the internet, making wp2shell one of the most consequential core vulnerabilities in years. The flaw was introduced in WordPress 6.9 with an update to the REST API batch endpoint, which allows multiple API requests to be processed in a single call. A specific sequence of a malformed batch route combined with a SQL injection payload allows an unauthenticated attacker — no account, no plugins, default installation — to escalate from API access to arbitrary PHP code execution on the server. WordPress 6.8 and earlier contain only the SQL injection component (not the RCE chain) and received a separate lower-severity fix in 6.8.6.

WordPress released emergency updates (6.9.5 and 7.0.2) on July 17 and activated forced automatic updates across the platform — a rare step reserved for the most severe vulnerabilities. Managed hosting providers (WP Engine, Kinsta, WordPress.com) typically push these automatically, but self-hosted sites with auto-updates disabled must act manually. No public exploit code has been released yet, but given the trivial attack surface and the volume of affected sites, weaponization is expected imminently.

The vulnerability's "pre-auth" nature is what makes it uniquely dangerous: there is no authentication bypass to chain, no credential to steal, no special endpoint to discover. Any WordPress site running a vulnerable version is exposed to every scanner on the internet.

🎓 For beginners: WordPress is software that powers millions of blogs, news sites, and online stores. This vulnerability means an attacker doesn't need a username or password — they can just send a specially crafted request to your WordPress site's built-in API and take over the entire server. Picture finding a hidden slot in a bank vault door that, if you insert the right oddly-shaped key anyone can make at home, opens the door without needing any account credentials.
CVE-2026-63030 · CVSS: 9.8/10 — Near-perfect score reflects unauthenticated network exploitation, low attack complexity, and critical impact on confidentiality, integrity, and availability of any default WordPress installation.

Attackers chain CVE-2026-58644 (unauthenticated RCE) with CVE-2026-56164 (missing-auth privilege escalation) to steal IIS machine keys, granting cryptographic control over all SharePoint-issued tokens and enabling persistent, credential-free re-entry into the environment.
▶ Full analysis & beginner explanation

Microsoft's record July 2026 Patch Tuesday (622 CVEs total) included a cluster of four actively exploited SharePoint vulnerabilities. The most critical is CVE-2026-58644 (CVSS 9.8): a deserialization of untrusted data flaw that lets an unauthenticated attacker send a crafted HTTP request to an on-premises SharePoint server and execute arbitrary code. CISA added it to the Known Exploited Vulnerabilities catalog July 17 with a binding deadline of today for federal civilian agencies. Despite Microsoft's surprising "Moderate" rating on CVE-2026-56164 (CVSS 5.3), researchers and vendors unanimously criticized the label — the actual exploitability is far higher when chained with other bugs.

In the current wave, attackers exploit CVE-2026-58644 for initial code execution, then use CVE-2026-56164 (missing authentication for SharePoint privilege functions) to steal the IIS machine keys. Possessing these cryptographic keys allows an adversary to forge authentication tokens, effectively owning the SharePoint environment indefinitely without needing any user credentials.

Organizations still running on-premises SharePoint should treat any unpatched server as potentially compromised. The same July Patch Tuesday also patched CVE-2026-56155 (ADFS privilege escalation, actively exploited) and CVE-2026-50661 (BitLocker encryption bypass, publicly disclosed).

🎓 For beginners: SharePoint is Microsoft's corporate file-sharing and collaboration platform used by governments and companies worldwide. CISA required government agencies to fix this today — any agency that missed the deadline not only faces a security breach, but also regulatory consequences.

ViteVenom: Seven Malicious npm Packages Use Blockchain C2 Across Three Chains to Deploy a Stealthy RAT on Developer Machines
🔍 What happened: Checkmarx researchers discovered ViteVenom, a supply chain attack campaign by threat actor SuccessKey that planted seven malicious npm packages impersonating the @vitejs/* namespace, using a four-tier blockchain-based command-and-control infrastructure spanning Tron, Aptos, and Binance Smart Chain to deliver a remote access trojan (RAT) capable of reverse shells, credential harvesting, file exfiltration, and persistent backdoor injection.
👤 Who's affected: JavaScript and frontend developers who work with the Vite build tooling ecosystem and may have installed any of the seven scoped packages published between June 29 and July 3, 2026 — particularly those who installed dependencies without verifying the exact package scope.
✅ What to do: Audit all npm dependencies for suspicious @vitejs-scoped packages installed in late June or early July; remove any flagged packages, rotate credentials and API keys stored in your development environment, and verify your CI/CD pipeline has not been compromised.
🎯 ATT&CK: [Initial Access] T1195.002 · Supply Chain: Compromise Software Dependencies
SuccessKey planted seven scoped npm packages impersonating the official @vitejs/* namespace, poisoning the developer supply chain so that any project installing a ViteVenom package silently installs a fully functional RAT on the developer's machine and build environment.
▶ Full analysis & beginner explanation

ViteVenom is the second campaign from threat actor SuccessKey, expanding on ChainVeil — an earlier operation notable for introducing four-tier blockchain-based C2 infrastructure to npm supply chain attacks. Unlike traditional C2 servers that can be seized by law enforcement or blocked by network defenders, blockchain transactions are immutable and globally distributed. The malware reads attacker instructions embedded in on-chain data across Tron, Aptos, and Binance Smart Chain — making the C2 channel virtually impossible to take down.

The seven malicious packages use scoped package names mimicking the official @vitejs organization (e.g., @vitejs/plugin-something-extra) to appear legitimate at a glance. A critical evasion twist: the malicious code does not execute at npm install time but only at import time — when a developer requires the package in their code. This bypasses many endpoint and CI security tools that monitor post-install scripts. Once imported, the RAT connects to the blockchain C2 and can receive commands for: interactive reverse shell access, harvesting credentials stored in browsers and configuration files, exfiltrating source code and secrets, and injecting persistent backdoor code into the developer's project.

Evidence of SuccessKey's blockchain wallet activity dates to February 27, 2026, suggesting months of infrastructure preparation before the packages were published June 29–July 3. This reflects a maturing attacker with patience and resources uncommon in typical supply chain campaigns.

🎓 For beginners: Developers use npm (Node Package Manager) to download pre-built code tools that save time. Attackers created fake packages with names that look almost identical to real, trusted tools (like changing "official-tool" to "0fficial-tool"). When a developer accidentally installs one, a hidden program (RAT) silently runs in the background. The clever twist: this RAT gets its instructions from the blockchain — the same technology behind Bitcoin — instead of a normal server, which means defenders cannot simply shut down a server to stop it.
No CVE — supply chain attack via malicious npm packages; blockchain C2 makes traditional takedown approaches ineffective.

GoSerpent is a sophisticated, long-dwell espionage implant written in Go — a language choice that eases cross-platform compilation and can complicate static signature analysis. Kaspersky's Global Research and Analysis Team (GReAT) uncovered the campaign in February 2026 and has now assessed it as active since at least 2021, making it a five-year intelligence-gathering operation. Rather than immediately exfiltrating files — which would trigger network anomaly detection — GoSerpent silently copies and stages sensitive data locally over extended periods. 💡 Today I Learned: Blockchain C2 ▶
Blockchain C2 (Command-and-Control) is a technique where malware retrieves its instructions not from a traditional server — which defenders can seize or block — but from transactions permanently recorded on public blockchains like Bitcoin, Ethereum, Tron, or Binance Smart Chain. The attacker simply publishes a small piece of encoded text in a blockchain transaction, and the malware reads it on its schedule; since blockchain records are immutable and decentralized, no single authority can remove the instructions or take down the "server." ViteVenom's use of a four-tier blockchain C2 spanning three separate chains represents the most sophisticated implementation seen to date, making takedown by any law enforcement agency or hosting provider effectively impossible as long as those blockchains operate.
