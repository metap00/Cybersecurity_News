### [Miasma Worm Hits 73 Microsoft GitHub Repositories in Major Supply Chain Attack](https://thehackernews.com/2026/06/miasma-worm-hits-73-microsoft-github.html)
**Source:** The Hacker News | **Published:** June 2026

A sophisticated supply chain worm known as Miasma has compromised 73 repositories across four of Microsoft's GitHub organizations — Azure, Azure-Samples, Microsoft, and MicrosoftDocs — in one of the most significant developer ecosystem attacks seen this year. GitHub disabled access to the affected repositories while Microsoft's investigation continues. The worm is assessed to be a variant of the Mini Shai-Hulud worm publicly released by TeamPCP in mid-May 2026.

Miasma does not exploit a technical flaw in npm or GitHub itself; instead, it abuses the trust model these platforms are built on. The worm planted a 4.3 MB payload runner and wired it to execute automatically through five developer tools: Claude Code, Gemini CLI, Cursor, VS Code, and npm test scripts. Any developer who opens an infected repository in an AI-powered coding tool or IDE risks having their credentials harvested — and the worm's propagation loop can then use those fresh tokens to compromise further repositories.

The attack highlights a dangerous new frontier in supply chain security: AI coding tools serve as both a propagation vector and an attack amplifier. Developers should audit any recently cloned Microsoft GitHub repositories, rotate their GitHub tokens, and avoid opening unfamiliar repositories in AI coding environments until the investigation concludes. Microsoft has begun restoring some repositories but has kept others offline as the probe continues.

### [Microsoft Defender 'RoguePlanet' Zero-Day (CVE-2026-50656) PoC Released — Patch Still in Development](https://www.bleepingcomputer.com/news/microsoft/microsoft-defender-rogueplanet-zero-day-grants-system-privileges/)
**Source:** Bleeping Computer | **Published:** June 2026

An anonymous security researcher going by "Chaotic Eclipse" (aka Nightmare-Eclipse) has publicly released a proof-of-concept exploit for a new Microsoft Defender zero-day named RoguePlanet, hours after Microsoft patched two separate Defender flaws during June Patch Tuesday. CVE-2026-50656 carries a CVSS score of 7.8 and allows attackers to spawn a command prompt with SYSTEM privileges on fully patched Windows 10 and Windows 11 systems by exploiting a race condition in Microsoft Defender.

The researcher reports achieving a 100% success rate on some test machines, though reliability varies. Critically, the exploit works on systems with June 2026 Patch Tuesday updates applied — meaning up-to-date Windows users are still vulnerable. Microsoft has confirmed the issue is under investigation and stated it is "working to provide a high-quality security update that addresses this vulnerability," but no patch is yet available.

These disclosures appear to be part of a retaliatory campaign following an alleged breakdown between the researcher and Microsoft's Security Response Center (MSRC). With a public PoC now circulating, organizations should monitor for privilege escalation activity on Windows endpoints, apply the patch immediately once it becomes available, and consider additional endpoint detection rules targeting SYSTEM-level process spawning from Defender components. The lack of an available fix makes this the most operationally dangerous disclosure of the day.

### [CISA Warns Critical Lantronix EDS5000 Flaw Actively Exploited — Federal Patch Deadline: Today](https://thehackernews.com/2026/06/cisa-warns-critical-lantronix-eds5000.html)
**Source:** The Hacker News | **Published:** June 2026

CISA issued an urgent warning about active exploitation of CVE-2025-67038, a critical code injection vulnerability (CVSS 9.8) in Lantronix EDS5000 Series device servers. The flaw allows unauthenticated remote attackers to execute arbitrary commands with elevated privileges. Federal Civilian Executive Branch (FCEB) agencies face a hard remediation deadline of June 26, 2026 — today — under CISA's Known Exploited Vulnerabilities (KEV) catalog mandates.

The EDS5000 series are industrial device servers used to connect serial devices (such as legacy equipment and sensors) to IP networks, commonly deployed in manufacturing, utilities, and critical infrastructure environments. The CVSS 9.8 score reflects the lack of authentication requirements and the full command-execution impact, making the attack surface extremely broad. Evidence of active in-the-wild exploitation means organizations cannot wait for a scheduled maintenance window.

Administrators should immediately apply Lantronix's security patch and, where immediate patching is not possible, isolate EDS5000 devices behind strict network segmentation to block external access. Organizations that have not yet remediated should treat this as a high-urgency emergency and review device logs for indicators of compromise consistent with command injection activity.

### [DragonForce Ransomware Gang Hides Backdoor C2 Traffic Inside Microsoft Teams Relays](https://thehackernews.com/2026/06/dragonforce-hackers-abuse-microsoft.html)
**Source:** The Hacker News | **Published:** June 2026

Researchers at Broadcom's Symantec and Carbon Black have uncovered a novel technique used by the DragonForce ransomware group: a custom Go-based remote access trojan called Backdoor.Turn that conceals its command-and-control (C2) traffic as legitimate Microsoft Teams communications. Backdoor.Turn obtains an anonymous Teams visitor token from Microsoft's Skype-backed identity services, uses a legitimate Microsoft TURN relay to establish its connection, and then runs a QUIC session to the attacker's real C2 server — all appearing to security products as normal Teams traffic.

This makes Backdoor.Turn the first known in-the-wild malware to abuse Microsoft Teams TURN relays for C2 communications, following a proof-of-concept demonstrated by Ghost Calls in 2025. The backdoor enables attackers to execute commands, create processes, perform network scanning and Active Directory reconnaissance, move laterally with stolen credentials, and exfiltrate browser-stored credentials. In a recent attack against a major U.S. services firm, DragonForce deployed this backdoor as part of a broader ransomware campaign.

Defenders should not rely solely on network monitoring tools that whitelist Microsoft Teams traffic. Organizations should review QUIC-over-TURN traffic patterns and monitor for anomalous Teams relay usage originating from non-standard processes. Endpoint detection rules looking for Go binaries making anonymous Teams authentication calls may help surface this threat. DragonForce has been active since 2023 and operates as a cartel structure, suggesting well-resourced adversaries with continued operational maturity.

### [Chrome V8 Zero-Day CVE-2026-11645 Exploited in the Wild — Update Your Browser Now](https://thehackernews.com/2026/06/chrome-v8-zero-day-cve-2026-11645.html)
**Source:** The Hacker News | **Published:** June 2026

Google has patched CVE-2026-11645 (CVSS 8.8), an out-of-bounds memory access vulnerability in Chrome's V8 JavaScript and WebAssembly engine, after confirming that an exploit for the flaw exists in the wild. The vulnerability allows a remote attacker to execute arbitrary code inside the browser sandbox via a crafted HTML page — in other words, visiting a malicious website is sufficient to trigger exploitation. Google awarded the discovering researcher, identified only as "303f06e3," a $55,000 bug bounty for responsible disclosure.

The fix is included in Chrome 149.0.7827.103. This marks the fifth actively exploited Chrome zero-day that Google has addressed since the start of 2026, following CVE-2026-2441, CVE-2026-3909, CVE-2026-3910, and CVE-2026-5281 — a pace that underscores the continued targeting of browser JavaScript engines by sophisticated threat actors. Google has withheld technical details to allow users to update before broader exploitation occurs.

Users and administrators should update Chrome immediately to version 149.0.7827.103 or later. Chromium-based browsers (Edge, Brave, Opera, etc.) will receive updates from their respective vendors and should also be patched promptly. Organizations should verify auto-update policies are functioning and consider temporarily restricting access to untrusted websites while patching is in progress.

### [Cisco SD-WAN Zero-Day CVE-2026-20245 Was Exploited Months Before Patch, Mandiant Reveals](https://www.securityweek.com/cisco-sd-wan-zero-day-exploited-months-before-patching/)
**Source:** SecurityWeek / BleepingComputer | **Published:** June 2026

New analysis from Google-owned Mandiant reveals that attackers exploited CVE-2026-20245, a high-severity command injection flaw in Cisco Catalyst SD-WAN Manager (CVSS 7.8), as a zero-day at least two months before Cisco publicly disclosed the vulnerability. Mandiant initially discovered the flaw while investigating attacks on a service provider's SD-WAN infrastructure between late 2025 and January 2026. Attackers exploited a tenant-upload feature in the SD-WAN CLI by uploading a malicious CSV file named "evil_tenant.csv," which created backdoor accounts — including one named "troot" — with full root-level privileges after copying /etc/passwd and /etc/shadow for offline analysis.

Cisco released final fixes on June 12, 2026. The timeline means organizations may have been compromised well before the vulnerability was publicly known, significantly complicating incident response. This is reportedly the seventh SD-WAN zero-day exploited in 2026 alone, reflecting sustained, targeted interest in SD-WAN infrastructure by sophisticated adversaries.

Organizations running Cisco Catalyst SD-WAN should apply the June 12 patches immediately if not already done, and conduct a thorough review of local accounts on SD-WAN devices for unauthorized entries. Particular attention should be paid to accounts created between October 2025 and the patch date. CISA has added CVE-2026-20262 (a related Cisco SD-WAN flaw) to the KEV catalog with a federal remediation deadline of June 29, 2026.

## Threat Landscape

Today's threat landscape is dominated by a convergence of supply chain attacks and living-off-the-land techniques that abuse trusted platforms. The Miasma worm's exploitation of AI coding tools as a propagation vector signals that developer environments — long considered a soft underbelly — are now primary targets for sophisticated adversaries. Simultaneously, DragonForce's abuse of Microsoft Teams relay infrastructure to hide ransomware C2 traffic demonstrates that attackers are systematically embedding themselves inside legitimate cloud services to evade detection.

Network device and enterprise software vulnerabilities remain a critical, actively exploited attack surface: Cisco SD-WAN, Splunk Enterprise, and Lantronix ICS devices were all under active exploitation this period, with some zero-days silently weaponized months before vendor disclosure. The Microsoft Defender RoguePlanet zero-day with a public PoC and no patch yet available raises the immediate risk for Windows environments. Organizations should prioritize patching Chrome, Lantronix devices, and Splunk Enterprise today, and treat AI-integrated developer workflows with the same scrutiny as production systems.
