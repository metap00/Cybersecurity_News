"Bad Epoll" Linux Kernel Zero-Day Lets Any User Grab Root — Public Exploit Hits 99% Success Rate
🔍 What happened: Researcher Jaeyoung Chung publicly disclosed CVE-2026-46242, a use-after-free race condition in the Linux kernel's epoll subsystem that allows any unprivileged local user to escalate to root — the exploit succeeds 99% of the time and works on Linux servers, desktops, and Android devices running kernel 6.4 or later.
👤 Who's affected: Any system running Linux kernel version 6.4 or later without the April 24 mainline patch, including most unpatched cloud servers, Linux workstations, and Android phones whose vendor has not yet shipped the fix.
✅ What to do: Apply your Linux distribution's kernel security update immediately; if your distro hasn't released a backport yet, monitor their security advisories closely and consider temporary mitigations such as restricting local user access on sensitive servers.
▶ Full analysis & beginner explanation

Bad Epoll targets the Linux kernel's epoll subsystem — the mechanism programs use to watch many file descriptors (open files, network sockets) for activity at once. The specific flaw is a use-after-free (UAF) in the ep_remove() cleanup function: the code clears a pointer under a lock but then continues referencing the now-freed memory object inside the same locked section. A concurrent kernel path called __fput() can race to free the underlying struct eventpoll at exactly this moment, corrupting kernel memory in a controlled way.

Researcher Jaeyoung Chung built a reliable exploit around this six-instruction-wide timing window, achieving root on 99% of attempts on tested kernelCTF targets. The attack requires only a standard unprivileged user account — no special hardware, no prior vulnerability chain. Strikingly, an Anthropic AI model (Mythos) successfully identified a companion race condition in the same 2,500-line commit that introduced Bad Epoll (now tracked as CVE-2026-43074), but missed this second flaw sitting directly adjacent to it, highlighting the current limits of AI-assisted code auditing.

The upstream kernel patch (commit a6dc643c6931) has been in mainline since April 24, 2026 — 70 days before the public writeup dropped — but the flaw was not publicly announced at that time. Many distribution vendors have not yet backported the fix, leaving production systems exposed despite the patch technically existing. CVE-2026-46242 is not currently on CISA's KEV catalog, and no confirmed in-the-wild exploitation has been reported yet. However, with a public proof-of-concept now available, the window before active exploitation is narrow.

🎓 For beginners: Think of the Linux kernel as the "engine room" of your computer — it controls everything beneath the apps you use. Bad Epoll tricks the engine into accidentally freeing a piece of memory it's still using, like pulling a support beam out from under a building while construction is ongoing. An attacker sitting at any regular user account on your Linux machine can use this trick to instantly become the all-powerful "root" administrator, with total control over every file, process, and secret on the system.

RoguePlanet (CVE-2026-50656): Microsoft Defender Zero-Day Grants SYSTEM Access — No Patch Yet
🔍 What happened: Security researcher Chaotic Eclipse published a working privilege escalation exploit for Microsoft Defender (CVE-2026-50656) that exploits a race condition in the Malware Protection Engine to hand any low-privileged attacker a shell with full SYSTEM-level privileges on any up-to-date Windows machine.
👤 Who's affected: All Windows users whose systems include Microsoft Defender — effectively the vast majority of Windows desktops and servers worldwide — with the exploit confirmed to work even when Defender's real-time protection is enabled and the system is fully patched.

▶ Full analysis & beginner explanation

RoguePlanet is a privilege escalation vulnerability in the Microsoft Malware Protection Engine — the core scanning component that runs as SYSTEM (the highest Windows privilege level) to inspect files and processes in real time. Researcher Chaotic Eclipse (aka Nightmare-Eclipse) found a race condition in this engine's scanning logic that can be won by a low-privileged attacker to hijack the engine's SYSTEM-level execution context and open a command shell with full administrative control.

The particularly alarming aspect of RoguePlanet is that it works because Defender is running, not despite it. Real-time protection must be active for the attack surface to exist, meaning disabling Defender's active scanning would technically prevent the exploit but also remove the system's primary malware defense. Microsoft confirmed the vulnerability after Chaotic Eclipse's public disclosure and stated that a patch is in development — but as of July 5, 2026, no fix has shipped.

This is the fourth zero-day in Microsoft Defender disclosed by the same researcher this year, following BlueHammer (CVE-2026-33825), UnDefend (CVE-2026-45498), and RedSun (CVE-2026-41091) — all three of which Microsoft has since patched. The pattern suggests a systematic audit of Defender's scanning engine internals by a single researcher with deep knowledge of its attack surface. Given the public PoC and high reliability, RoguePlanet should be treated as actively exploitable until a patch arrives.

🎓 For beginners: Microsoft Defender is the built-in antivirus on Windows that runs as the most privileged user on the computer to scan everything. RoguePlanet turns Defender's own power against you — by racing to slip malicious instructions in at the precise moment Defender is doing its most powerful work, an attacker with only a basic user account can hijack Defender's privileges and take full control of the computer. It's like pickpocketing the security guard's master key while he's patting you down.

💡 Today I Learned: Kernel Privilege Escalation ▶
The kernel is the deepest, most powerful layer of an operating system — it runs as the highest-privileged user on the machine and directly controls hardware, memory, and every process running on the computer. Privilege escalation is when an attacker who starts with a limited, low-powered account finds a bug that lets them "level up" their permissions to root or SYSTEM — the all-powerful administrator level — even though no one authorized that upgrade. Today's "Bad Epoll" zero-day is a textbook example: by exploiting a tiny timing flaw in kernel cleanup code, an ordinary user with no special access can become root with 99% reliability, turning a limited guest account into unlimited control over the entire machine.
