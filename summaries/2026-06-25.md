### ['Cordyceps' CI/CD Flaw Exposes 300+ GitHub Repos to Supply-Chain Attacks](https://thehackernews.com/2026/06/cordyceps-cicd-flaws-expose-300-github.html)
**Source:** The Hacker News | **Published:** June 24, 2026

Novee Security disclosed a critical class of CI/CD workflow weakness—codenamed "Cordyceps"—that allows any unauthenticated attacker to hijack GitHub Actions workflows and compromise open-source software supply chains at scale. A scan of roughly 30,000 high-impact repositories found more than 300 to be fully exploitable, with affected organizations including Microsoft, Google, Apache, Cloudflare, and the Python Software Foundation.

The core flaw is a misconfiguration in which low-privilege CI workflows pass their output to high-privilege workflows, creating an escalation chain that ultimately grants owner-level cloud-provider access. The vulnerability is exploitable with nothing more than a free GitHub account—no org membership or special privileges required—allowing an attacker to forge PR approvals, inject code into production pipelines, and steal non-expiring GitHub App credentials. In a live demonstration on Microsoft's Azure Sentinel repository, Novee researchers showed that a comment on a pull request could trigger code execution on Microsoft's own CI runners.

Security researchers warn that the problem is self-replicating: AI-assisted code generation tools reproduce the same insecure workflow patterns across repositories, causing the vulnerability to propagate at exponential rates as more developers adopt agentic coding workflows. Affected organizations should audit their GitHub Actions configurations, enforce minimum-privilege workflow tokens, and disable pull-request-triggered write access for untrusted forks.

### [Cisco Unified CM Webshell Drops Confirmed — Patch Alone Won't Evict Attackers (CVE-2026-20230)](https://www.helpnetsecurity.com/2026/06/24/cisco-unified-cm-flaw-exploited-to-drop-webshells-cve-2026-20230/)
**Source:** Help Net Security | **Published:** June 24, 2026

Active exploitation of CVE-2026-20230, an improper input-validation flaw (CVSS 8.6) in Cisco Unified Communications Manager's WebDialer service, has escalated dramatically. Attackers have progressed from weekend reconnaissance to automated Tor-routed deployment of three-stage JSP command shells as of June 24, 2026. The vulnerability allows unauthenticated remote attackers to conduct server-side request forgery (SSRF) attacks and write files to the underlying OS, enabling subsequent privilege escalation to root.

Cisco released a patch on June 3, 2026, but security responders warn that simply patching is insufficient where webshells have already been implanted—organizations must also conduct forensic sweeps for JSP files in web-accessible directories, rotate credentials, and treat any affected CM instance as potentially fully compromised. The WebDialer service—the vulnerable component—is disabled by default; organizations with it enabled are at immediate risk.

Affected products include all supported versions of Cisco Unified CM and Unified CM Session Management Edition. Security teams should apply Cisco's June 3 fix immediately, verify WebDialer is disabled unless explicitly required, and inspect server file systems for backdoors.

### [Xsolis Healthcare Data Breach Exposes Data of 1.4 Million Patients](https://www.helpnetsecurity.com/2026/06/24/xsolis-data-breach-phishing-attack/)
**Source:** Help Net Security | **Published:** June 24, 2026

Healthcare AI platform Xsolis disclosed a data breach affecting approximately 1.4 million individuals following a targeted phishing attack that compromised its network in January 2026. The company, which provides AI-powered software to more than 600 hospitals, health systems, and health insurers—including Humana—confirmed that attackers moved from initial phishing compromise to active data exfiltration within 48 hours, a timeline characterizing this as a targeted rather than opportunistic attack.

Exposed data may include names, addresses, dates of birth, health insurance information, Social Security numbers, and medical treatment information, depending on the affected individual. Seven health systems have confirmed patient data was involved. Xsolis detected the breach on January 22, 2026 and immediately engaged external cybersecurity experts to contain it; the company has found no evidence of data misuse as of the public disclosure.

Law firm Edelson Lechtzin has launched an investigation on behalf of affected individuals, and multiple class-action suits are expected. Affected patients should monitor credit reports and health insurance explanations of benefits for suspicious activity, and any organization sharing data with Xsolis via API integrations should review those connections.

### [CISA Adds Lantronix EDS5000 and Ubiquiti UniFi OS Flaws to KEV Catalog — Federal Deadline June 26](https://securityaffairs.com/194142/security/u-s-cisa-adds-ubiquiti-unifi-os-and-lantronix-eds5000-plugin-flaws-to-its-known-exploited-vulnerabilities-catalog.html)
**Source:** Security Affairs | **Published:** June 23, 2026

CISA added four actively exploited vulnerabilities to its Known Exploited Vulnerabilities (KEV) catalog on June 23, 2026, with a Federal Civilian Executive Branch (FCEB) remediation deadline of June 26, 2026. The most critical is CVE-2025-67038 (CVSS 9.8) in Lantronix EDS5000 Series serial-to-IP console servers—a code injection vulnerability in the HTTP RPC module that allows unauthenticated attackers to execute arbitrary OS commands as root by injecting into an unsanitized `username` parameter during failed authentication logging.

Affected models include the EDS5008, EDS5016, and EDS5032 running firmware 2.1.0.0R3. Lantronix has released firmware 2.2.0.0R1 to close the vulnerability. CISA also added flaws in Ubiquiti UniFi OS, which reached max-severity scores and are likewise under active exploitation.

These devices are commonly deployed in industrial control environments, data centers, and network operations centers, making exploitation particularly dangerous for critical infrastructure operators. All organizations—not just federal agencies—should treat the June 26 deadline as a benchmark for emergency patching.

### [Klue Supply Chain Breach: Icarus Group's Salesforce OAuth Theft Hits Major Cybersecurity Firms](https://techcrunch.com/2026/06/22/klue-hack-results-in-data-breach-at-several-cybersecurity-firms/)
**Source:** TechCrunch / BleepingComputer | **Published:** June 22–25, 2026 (ongoing)

A sophisticated supply chain attack against Vancouver-based market intelligence platform Klue has resulted in data breaches at dozens of organizations, including prominent cybersecurity vendors. The newly emerged Icarus extortion group compromised Klue on June 11–12 using a legacy integration service account credential, then pushed a malicious code update that harvested OAuth tokens—the authorization keys Klue uses to connect with customers' Salesforce CRM instances.

Confirmed victims include Recorded Future, Tanium, HackerOne, Snyk, Jamf, Gong, OneTrust, Insurity, Sprout Social, and Kudelski Security. Icarus has threatened to publish stolen data unless ransoms are paid, and the victim list continues to grow as more organizations complete their impact assessments. Klue engaged CrowdStrike for incident response and has notified law enforcement.

The breach illustrates the compounding risk of SaaS supply-chain attacks: a single compromised integration layer granted access to CRM data across hundreds of enterprise customers. Security teams should immediately audit third-party OAuth grants in Salesforce and revoke any Klue-related tokens, monitor for anomalous CRM exports, and review least-privilege policies for all SaaS integrations.

### [Chrome V8 Zero-Day CVE-2026-11645 Actively Exploited — Update to Version 149 Now](https://thehackernews.com/2026/06/chrome-v8-zero-day-cve-2026-11645.html)
**Source:** The Hacker News | **Published:** June 9–23, 2026

Google patched a high-severity out-of-bounds memory access vulnerability (CVE-2026-11645, CVSS 8.8) in Chrome's V8 JavaScript engine—the fifth Chrome zero-day exploited in 2026—and CISA added it to the KEV catalog with a remediation deadline of June 23. The flaw can be triggered when a user visits a specially crafted HTML page, potentially allowing remote code execution within the browser sandbox and enabling heap corruption that could help bypass ASLR protections.

The vulnerability affects all Chromium-based browsers, including Google Chrome, Microsoft Edge, Brave, and Opera. Google has confirmed an exploit exists in the wild but has not disclosed victimology or technical exploit details. The fixed versions are 149.0.7827.102/.103 for Windows and macOS, and 149.0.7827.102 for Linux. All users of Chromium-based browsers should update immediately; enterprise administrators should force-push the update through their management platforms.

## Threat Landscape

Today's threat picture is defined by supply chain risk on two fronts: the Cordyceps CI/CD disclosure reveals systemic vulnerabilities in how AI-generated workflow configurations propagate insecure patterns across millions of repositories, while the Klue-Salesforce OAuth breach demonstrates how a single integration layer can cascade breaches across dozens of enterprise victims. Active exploitation of network infrastructure flaws—particularly Cisco UCM, Lantronix serial-IP servers, and Ubiquiti UniFi—highlights continuing attacker focus on persistent access in IT/OT convergence points. Healthcare remains a high-value target, as the Xsolis breach shows attackers are conducting rapid, targeted intrusions designed to exfiltrate sensitive patient data before defenses can respond.
