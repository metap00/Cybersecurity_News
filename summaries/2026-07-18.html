15-Year-Old GhostLock Flaw Enables Root and Container Escape on Most Linux Distros
🔍 What happened: A 15-year-old use-after-free bug in the Linux kernel's real-time mutex code (CVE-2026-43499) now has a public, 97%-reliable proof-of-concept that gives any local user full root access in roughly five seconds and also escapes containers.
👤 Who's affected: Every Linux distribution running kernels 2.6.39 (2011) through 7.0 — including Ubuntu LTS releases, Debian, RHEL, and multi-tenant cloud and Kubernetes environments.
✅ What to do: Update to a distribution-patched kernel or Linux 7.1 immediately; restrict local login access on shared systems; cloud providers must patch host kernels to protect tenant workloads.
🎯 ATT&CK: [TA0004 - Privilege Escalation] T1068 · Exploitation for Privilege Escalation
The GhostLock use-after-free in the rtmutex priority-inheritance code is weaponized to overwrite adjacent kernel objects and gain ring-0 execution — the textbook exploitation-for-privilege-escalation pattern.
▶ Full analysis & beginner explanation
GhostLock exploits a use-after-free condition in the Linux kernel's real-time mutex priority-inheritance subsystem, first introduced in kernel 2.6.39 in 2011. When multiple threads race to acquire and release priority-inherited locks under carefully timed conditions, a dangling pointer to a freed rtmutex object can be manipulated to overwrite adjacent kernel data structures. The exploit requires no elevated permissions — ordinary threading syscalls from any unprivileged process are sufficient. Nebula Security's VEGA team disclosed the bug on July 7 alongside a full technical write-up and open-source PoC code that achieves 97% reliability in testing. Google's kernelCTF program awarded a $92,337 bounty for the report.

The most alarming dimension is container escape: the same exploit, executed from inside a Docker or Kubernetes container, breaks through the isolation boundary to gain root on the underlying host — affecting every other tenant workload on that machine. Ubuntu 24.04, 22.04, and 20.04 LTS were still listed as partially unpatched as of the disclosure date. Enterprise Linux vendors have released patches, but rollout across heterogeneous fleets takes time, leaving a large attack surface for insiders, compromised CI/CD agents, or attackers who gain any foothold in a shared environment.
🎓 For beginners: Imagine the Linux kernel is the master key-control room of a building. GhostLock tricks the room into handing a master key to any random employee by exploiting a 15-year-old bookkeeping error in how shared locks are managed. Once you have that master key, you can open every door — including the walls separating different companies' suites in the same building (that's the container escape).
CVE-2026-43499 · CVSS: 7.8/10 — High severity local privilege escalation; CVSS reflects local-only access requirement, but the public 97%-reliable PoC and container-escape capability elevate real-world risk to critical.

New wp2shell WordPress Core Flaw Lets Unauthenticated Attackers Run Code on 500M Websites
🔍 What happened: A chained pre-authentication RCE dubbed wp2shell (CVE-2026-63030 + CVE-2026-60137) was found in WordPress core — no plugins required — allowing any anonymous internet user to execute arbitrary code on any unpatched WordPress site via REST API batch route confusion combined with SQL injection.
👤 Who's affected: All WordPress installations running versions 6.9.0–6.9.4 and 7.0.0–7.0.1, estimated at 500+ million websites including major news outlets, e-commerce platforms, and enterprise portals.

🎯 ATT&CK: [TA0001 - Initial Access] T1190 · Exploit Public-Facing Application
An unauthenticated HTTP request to the WordPress REST API batch endpoint exploits route confusion and SQL injection to achieve remote code execution, directly targeting the public web application interface with no credential requirement.
▶ Full analysis & beginner explanation
The wp2shell chain combines two weaknesses: CVE-2026-63030, a route-confusion bug in the WordPress REST API's batch processing endpoint that misroutes requests to unintended internal handlers, and CVE-2026-60137, an SQL injection in the author__not_in parameter of WP_Query that can be triggered without authentication through the confused route. Chained together, an anonymous HTTP request can inject arbitrary SQL, ultimately reaching file-write primitives and OS-level command execution on the web server. The attack works on a fresh stock WordPress install — no plugins, no unusual configuration, no credentials needed.

Adam Kues of Assetnote / Searchlight Cyber discovered and responsibly disclosed the chain, allowing WordPress to ship patches (7.0.2 and 6.9.5) on July 17 before publishing technical details. A public exposure checker is available at wp2shell.com. Given that approximately half the world's websites run WordPress, and the batch REST API is enabled by default, the attack surface is enormous. Security teams should treat unpatched WordPress installs as actively compromised until updated.

💡 Today I Learned: Container Escape ▶
Containers (like Docker or Kubernetes pods) isolate processes by using Linux kernel features — namespaces and cgroups — to make each container think it's alone on the machine, even though many share the same underlying host kernel. A container escape is when an attacker running code inside a container exploits a vulnerability — usually in the host kernel itself — to break through that isolation boundary and gain access to the underlying host operating system and all other containers on the same machine. The GhostLock flaw is a textbook container escape: because all containers share the same kernel, a kernel-level bug that gives root access inside one container simultaneously grants root access to the entire host and every other tenant workload running on it.
📖 Glossary: EoP · Vishing · OOB Write · DoS · SSO ▶
EoP — Elevation of Privilege; a vulnerability class where an attacker with limited access exploits a flaw to gain higher-level system rights, such as local administrator, SYSTEM, or domain admin.
Vishing — Voice phishing; a social-engineering attack where criminals impersonate trusted parties (IT support, banks, executives) by telephone to trick targets into revealing credentials or resetting security controls like MFA.
OOB Write — Out-of-Bounds Write; a memory corruption flaw where software writes data past the end of an allocated buffer, potentially allowing an attacker to overwrite adjacent memory structures and gain code execution or privilege escalation.
DoS — Denial of Service; an attack that makes a system, service, or network resource unavailable to legitimate users by overwhelming it with traffic or exploiting a flaw that causes a crash or excessive resource consumption.
SSO — Single Sign-On; an authentication scheme that allows users to log in once and access multiple applications without re-entering credentials; Active Directory Federation Services (ADFS) is Microsoft's enterprise SSO platform, making ADFS zero-days particularly high-value for attackers targeting federated environments.
