18-Nation Advisory: Russian FSB Center 16 Systematically Compromising Routers to Infiltrate Critical Infrastructure
🔍 What happened: The NSA, CISA, FBI, and 15 partner agencies from 9 countries issued a joint advisory confirming that Russian FSB Center 16 (Berserk Bear) is actively exploiting weak SNMP configurations on internet-exposed routers to steal network topology data and establish persistent footholds in critical infrastructure worldwide.
👤 Who's affected: Organizations across defense, energy, financial services, government, healthcare, and communications sectors — particularly those using default SNMP community strings or running Cisco Smart Install.
✅ What to do: Immediately disable Cisco Smart Install, migrate all routers to SNMPv3 with unique credentials, block TFTP and SNMP traffic at edge firewalls, and audit router configurations for unauthorized changes or unknown TFTP transfers.
🎯 ATT&CK: [Collection] T1602.001 · Data from Configuration Repository: SNMP MIB Dump
FSB Center 16 uses SNMP with default or guessed community strings to issue commands that copy complete router configuration files via TFTP to attacker-controlled servers, exposing full network topology, routing tables, credentials, and pivot points.
▶ Full analysis & beginner explanation
Russia's FSB Center 16 — tracked in the threat intelligence community as Berserk Bear, Energetic Bear, Dragonfly, and Ghost Blizzard — has been running a sustained campaign targeting internet-exposed routers that accept weak or default SNMP authentication. SNMP (Simple Network Management Protocol) is used to monitor and manage network equipment, but version 1 and 2c rely on simple plaintext "community strings" as passwords. Attackers scan IP ranges for routers accepting strings like "public," "private," or "community," then use those strings to pull complete configuration dumps via TFTP.

The stolen configuration files are a goldmine: they reveal the organization's entire network architecture, internal IP address ranges, routing rules, access control lists, and sometimes include credentials for VPNs and other services in plaintext. Armed with this information, attackers can plan highly targeted intrusions, identify the most valuable assets, and exploit trust relationships between network segments. In some campaigns, the group has gone further by loading modified router firmware to establish backdoors that survive reboots and factory resets.

The 18-nation signature on this advisory is exceptional — it signals a coordinated intelligence assessment that this campaign is significantly escalating. Berserk Bear has historically focused on pre-positioning within critical infrastructure, and past campaigns have targeted power grids in Europe and North America. Geopolitical analysts note that such pre-positioning campaigns typically precede attempts at sabotage during periods of heightened tension.
🎓 For beginners: Your internet router is like the front door of your network — and it can be configured to let network administrators manage it remotely. FSB Center 16 (hackers working for Russia's intelligence service) are scanning the entire internet for routers with weak or default passwords on that remote management feature. When they find one, they quietly copy the router's "instruction manual" — a file that reveals exactly how the network is built — so they can plan break-ins or potentially cause outages later.
No software CVE — exploitation relies on misconfiguration (default/weak SNMP community strings), not an unpatched software flaw.

CISA KEV: Joomla Zero-Days CVE-2026-48939 & CVE-2026-56291 Exploited in Automated Attacks Since June
🔍 What happened: CISA added two CVSS 10.0 file-upload vulnerabilities in popular Joomla extensions to its KEV catalog — CVE-2026-48939 (iCagenda event plugin) and CVE-2026-56291 (Balbooa Forms builder) — with iCagenda exploitation confirmed in automated zero-day attacks since June 15, 2026.
👤 Who's affected: Any Joomla website using the iCagenda or Balbooa Forms extensions — widely used by small businesses, nonprofits, government sites, and educational institutions globally.
✅ What to do: Update iCagenda to version 4.0.8 or 3.9.15 and Balbooa Forms to version 2.4.1 immediately; if patching is delayed, disable file upload functionality in both plugins as a temporary mitigation.
🎯 ATT&CK: [Persistence] T1505.003 · Server Software Component: Web Shell
Attackers exploit the unrestricted file upload flaw in both Joomla extensions to upload PHP web shells, gaining persistent server-side remote code execution on victim web servers and enabling follow-on data theft, ransomware staging, or cryptomining.
▶ Full analysis & beginner explanation
Two vulnerabilities in widely-deployed Joomla extensions have been weaponized as zero-days. CVE-2026-48939 in the iCagenda event calendar plugin allows unauthenticated attackers to upload arbitrary files — including PHP scripts — through the plugin's event attachment feature, which lacks file type validation. CVE-2026-56291 in Balbooa Forms exploits the same class of vulnerability through the forms plugin's file input fields. Both carry CVSS scores of 10.0 — the highest possible rating — reflecting that they require no privileges and grant full server control.

CISA's addition of both to the KEV catalog on July 13 — with the same day as the federal patching deadline — signals extreme urgency. CVE-2026-48939 has been confirmed in automated mass-scanning attacks against Joomla sites since June 15, meaning attackers have had nearly a month of unpatched exploitation time. Automated scanners identify Joomla installations, check for the vulnerable plugins, then upload PHP web shells that provide persistent remote execution on the web server.

Web shells uploaded through these flaws can persist even after plugin removal unless the uploaded files are specifically found and deleted from the server's file system. Organizations should scan their web roots for unexpected PHP files, review server logs for unusual file upload requests, and check for indicators of compromise before applying patches.
🎓 For beginners: Joomla is a popular website-building platform. These two plugins had a critical flaw: they let anyone upload any type of file without checking what it was. Attackers used this to upload a "web shell" — a hidden program that gives them remote control over the entire web server, like leaving a skeleton key inside someone's house. Since June, automated bots have been scanning the entire internet for Joomla sites with these plugins and exploiting them automatically.

✅ What to do: Upgrade to Zimbra Collaboration Suite 10.1.19 immediately; if immediate patching is not possible, switch users to the Modern Web Client as a temporary workaround until the patch is applied.
🎯 ATT&CK: [Execution] T1059.007 · Command and Scripting Interpreter: JavaScript
The stored XSS payload is embedded in a crafted email and automatically executes JavaScript within the victim's authenticated browser session on email open, enabling session token theft, account takeover, or delivery of additional malware without any additional user interaction.
▶ Full analysis & beginner explanation
Zimbra Collaboration Suite's Classic Web Client contains a critical stored XSS vulnerability that allows an attacker to craft a malicious email that automatically executes JavaScript code in the recipient's browser when the message is opened or previewed. Unlike reflected XSS attacks — which require the victim to click a suspicious link — stored XSS payloads persist within the email server and fire automatically for every user who views the message, including on reply chains and forwarded copies.

The vulnerability was discovered and reported by Google's Threat Analysis Group (TAG), which exclusively tracks state-sponsored attackers. TAG's involvement is a significant escalation signal: the group has previously discovered identical Zimbra flaws actively exploited by Russian groups (Winter Vivern, APT28) and North Korean actors to steal email credentials from NATO government ministries, military organizations, and diplomatic missions. While public exploitation has not been confirmed for this specific flaw, TAG typically discloses vulnerabilities only after verifying attacker interest or active use.

A successful attack could enable full email account takeover, exfiltration of sensitive communications, and lateral movement by hijacking authenticated sessions to reach connected systems. For government and intelligence targets — common Zimbra users — this represents a particularly high-value attack vector given the sensitivity of communications stored in enterprise email systems.
🎓 For beginners: Zimbra is an email platform used by governments and companies around the world. This flaw lets an attacker send you a specially crafted email that secretly runs a program in your browser the moment you open it — without you clicking anything suspicious. The fact that Google's team that specifically tracks government hackers found this vulnerability means it was likely being used or prepared for use by a nation-state spy group to steal sensitive government emails.

Forg365: New $400/Month PhaaS Combines AI Lures, Device Code Phishing & AitM to Bypass MFA on Microsoft 365
🔍 What happened: Researchers disclosed Forg365, a new Phishing-as-a-Service platform sold on Telegram for $400/month that combines AI-generated phishing lures, device code flow abuse, and adversary-in-the-middle session interception to bypass MFA and hijack Microsoft 365 accounts at scale.
👤 Who's affected: Microsoft 365 users at any organization that hasn't restricted device code flow authentication — enterprises, government agencies, universities, and mid-market businesses are all exposed.
✅ What to do: Block device code flow in Entra ID Conditional Access policies unless strictly required; require phishing-resistant MFA (FIDO2 hardware keys); monitor for unexpected OAuth token grants; alert on ForgCookie Chrome extension installation.
🎯 ATT&CK: [Credential Access] T1528 · Steal Application Access Token
Forg365 abuses Microsoft's OAuth 2.0 Device Authorization Flow to trick victims into entering a legitimate device code that authorizes an attacker-controlled session — stealing valid access tokens without ever needing the user's password or MFA code.
▶ Full analysis & beginner explanation
Forg365 is a sophisticated Phishing-as-a-Service platform sold through Telegram for $400/month or $3,800/year, targeting Microsoft 365 using two complementary attack vectors. The device code phishing branch presents victims with a Microsoft-styled page using a legitimate Microsoft Authentication Broker flow — so users see real Microsoft sign-in surfaces — but the authorization code they enter activates an attacker-controlled OAuth session rather than their own device. The AitM (adversary-in-the-middle) branch sits transparently between victim and real Microsoft login pages, capturing session cookies in real time to bypass MFA entirely.

The platform includes antibot detection (redirecting VPN traffic to harmless decoy pages), AI-assisted lure generation using legitimate email delivery infrastructure like Amazon SES and Twilio SendGrid, and a Chrome browser extension named ForgCookie for persistent post-compromise account access. Most concerning is its post-compromise capability: operators can configure keyword monitoring across compromised inboxes and use AI to draft contextually appropriate responses within victim email threads — enabling highly convincing spear-phishing of colleagues from the victim's own trusted email address.

The $400/month price point means even low-skilled criminal actors can now run sophisticated, MFA-bypassing attacks at scale. The commoditization of these capabilities follows a trend of once-advanced attack techniques becoming accessible to a much broader criminal ecosystem.
🎓 For beginners: Microsoft has a login method designed for smart TVs and devices without keyboards — it shows you a code on screen and asks you to type it on your phone to approve the login. Forg365 tricks you into using that code to approve an attacker's login session instead of your own. You see a real Microsoft page, you complete real Microsoft authentication steps, but you've just unlocked the attacker's door rather than your own. Even if you have two-factor authentication turned on, this attack can bypass it completely.
No CVE — Forg365 exploits legitimate OAuth 2.0 protocol features (Device Authorization Flow) abusively rather than a software vulnerability.
💡 Today I Learned: Phishing-as-a-Service (PhaaS) ▶
Phishing-as-a-Service (PhaaS) works like a subscription software toolkit — for a monthly fee, criminals can buy ready-made phishing pages, AI-crafted email lures, and automated attack infrastructure without needing any hacking skills themselves, the same way a business buys accounting software instead of hiring a programmer. Just as Software-as-a-Service lowered the bar for building legitimate applications, PhaaS platforms like today's Forg365 have democratized sophisticated credential theft, meaning even low-skilled actors can now run large-scale campaigns that bypass multi-factor authentication. The business model mirrors legitimate SaaS — tiered pricing, regular feature updates, and customer-support channels — turning what was once elite hacker tradecraft into a commodity anyone with $400/month can access.
📖 Glossary: PhaaS · AitM · SNMP · FSB · Device Code Flow · Storage Zone Controller ▶
PhaaS (Phishing-as-a-Service) — A subscription-based criminal business model that sells ready-made phishing toolkits — including fake login pages, email templates, and automation infrastructure — to attackers who lack the technical skills to build them from scratch.
AitM (Adversary-in-the-Middle) — An attack where the adversary sits transparently between a victim and a legitimate service, relaying login traffic in real time to capture session cookies and authentication tokens even after the victim has completed MFA.
SNMP (Simple Network Management Protocol) — A protocol used to remotely monitor and manage network devices like routers and switches; older versions (v1 and v2c) use weak plaintext "community strings" as passwords, making them vulnerable to exploitation if left at default values.
FSB (Federal Security Service) — Russia's primary domestic intelligence and counterintelligence agency, successor to the KGB, which also operates significant offensive cyber capabilities; FSB Center 16 is the unit responsible for targeted infrastructure attacks.
Device Code Flow — An OAuth 2.0 authentication method designed for devices without keyboards (smart TVs, IoT devices) that displays a short code users must enter on a separate device; abused in phishing attacks to trick victims into authorizing attacker-controlled sessions.
Storage Zone Controller — A Citrix ShareFile component that organizations install on their own servers so files remain on local infrastructure while still using ShareFile's cloud platform for management and sharing; internet-exposed by design, making it a target for external attacks.
